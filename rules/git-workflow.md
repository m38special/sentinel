# Git Workflow Rules

## Commits
- Conventional commits format
- `feat:`, `fix:`, `docs:`, `chore:`
- Present tense: "add" not "added"

## Branches
- main — Production
- phaseN — Feature branches
- hotfix/* — Quick fixes

## PR Process
1. Branch from main
2. Make changes
3. Run tests
4. Open PR
5. Get 3 AI reviews
6. Human merge

## Before Push
- [ ] Tests pass
- [ ] No console.logs
- [ ] No hardcoded secrets
- [ ] Code formatted
